# 使用路线与关键梳理（全自动流水线版）

一、各文件的关键（一句话说明，谁读谁更新）：
- **CONTRACTS.md**：API 和数据契约。由需求解读 Agent 起草，规则助手自动写入。
- **TASKS.md**：任务定义。由需求解读 Agent 产出，规则助手自动写入。
- **STATUS.md**：施工日志。由前端/后端 Agent 在完工后自动追加。
- **meta/DECISIONS.md**：决策历史。由人手动记录，所有 Agent 必读。
- **meta/BOARD.md**：任务看板。由规则助手、前端/后端 Agent 共同自动更新。
- **meta/CHECK_REPORT.md**：测试报告。由测试 Agent 在“待验收”阶段自动追加。

二、全自动流水线模式（路线 A）：

1. **需求拆解**：您将原始需求发给“需求解读 Agent”。
2. **人审确认**：您审核产出的文案，确认无误后说：“同意，执行”。
3. **自动同步**：规则助手（我）根据指令，自动更新 `TASKS.md`、`CONTRACTS.md` 和 `meta/BOARD.md`。
4. **主动领活**：前端/后端 Agent 读到 BOARD 更新，自动开分支领活，做完自动写 `STATUS.md` 并改 BOARD 为“待验收”。
5. **自动验证**：测试 Agent 读到“待验收”，自动去跑验证，产出 `meta/CHECK_REPORT.md`。
6. **人最后看一眼**：通过，合并。

三、一句话记住这套流程：
您只需在“需求拆解”后说一句“同意，执行”，剩下的任务定义、派活、领活、记录、验证全部由 Agent 自动流转，您只负责最后的验收合并。
