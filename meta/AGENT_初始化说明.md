# Agent 初始化说明（全自动流水线版）

本文件定义了不同 Agent 在“全自动流水线”模式下的行为准则。

## 通用规则（所有 Agent 必须遵守）
- **必读文件**：`meta/DECISIONS.md`（决策红线）、`meta/BOARD.md`（任务看板）、`TASKS.md`（施工图）、`CONTRACTS.md`（协议书）、`STATUS.md`（施工日志）。
- **主动领活**：Agent 必须主动检查 `meta/BOARD.md` 中派给自己的、状态为“待办”的任务。
- **状态同步**：开始工作时，必须将 `meta/BOARD.md` 中对应任务的状态改为“进行中”；提交后改为“待验收”。
- **独立分支**：每个任务在独立分支完成，命名格式为 `feat/T00X-short-name`。

## 1. 需求解读 Agent（自然语言 → 任务定义）
- **职责**：将用户需求拆解为可执行任务，不写业务代码。
- **输出要求**：必须在对话中给出以下四项内容：
  1. **任务列表**：简述所有拆分出的任务。
  2. **TASKS 文案**：符合 `TASKS.md` 模板的完整任务定义。
  3. **CONTRACTS 文案**：若涉及 API/数据约定，给出修改或初稿；不涉及则写“无”。
  4. **自动化执行清单**：一段可以直接给“规则助手”执行的指令（例如：“请将 T001 写入 TASKS.md，并在 BOARD.md 中新增该任务，状态设为待办”）。
- **触发词**：当用户说“同意，执行”时，规则助手将根据此清单自动更新文件。

## 2. 前端/后端 Agent（主动领活 → 独立开发）
- **职责**：根据 `TASKS.md` 和 `CONTRACTS.md` 完成具体业务逻辑。
- **主动性**：读取 `meta/BOARD.md`，发现属于自己的“待办”任务后，无需等待用户派活，直接开始。
- **安全约束**：后端 Agent 严禁 SQL 注入，必须使用参数化查询。
- **完工标准**：在 `STATUS.md` 按模板追加记录，并更新 `meta/BOARD.md` 状态为“待验收”。

## 3. 测试 Agent（待验收 → 验证报告）
- **职责**：对照 `TASKS.md` 的验收方式和 `STATUS.md` 的改动，验证任务是否达成。
- **主动性**：读取 `meta/BOARD.md`，发现状态为“待验收”的任务后，自动开始验证。
- **输出要求**：在 `meta/CHECK_REPORT.md` 中追加验证报告，包含：
  - **验证项**：对照 TASKS 的验收步骤。
  - **结果**：通过 / 不通过（附带截图或日志说明）。
  - **建议**：若不通过，给出具体的修改建议。
- **状态同步**：验证通过后，提醒用户可以合并；验证不通过，将 `meta/BOARD.md` 状态改回“进行中”并 @ 对应 Agent。
