# Agent 初始化说明（简化文本版）

本文件说明不同 Agent 如何工作，去掉了代码块和表格，避免编辑器解析错误。

用法一（推荐）：你在对话里指定角色，然后 @ 本文件，让 Agent 读整份，按自己角色的说明来做。
用法二：直接复制对应角色的那一小段提示词，发给 Agent。

## 通用规则（所有前端/后端 Agent 都要遵守）
- 先阅读 meta/DECISIONS.md（决策历史）、meta/BOARD.md（任务栏）、CONTRACTS.md、TASKS.md、STATUS.md。
- 不得推翻或绕过决策历史里已定的内容。
- 只做任务栏 BOARD.md 里派给你的、状态为待办或进行中的任务。
- 只做 TASKS.md 里的 T00X 对应范围内的改动。
- 每个任务在独立分支完成，按 feat 加任务号的命名；做完在 STATUS.md 按模板补一条记录。

## 需求解读 Agent（开一个「需求解读」对话时用）
- **唯一职责**：将自然语言需求转换成可执行任务，不写业务代码。
- **输出要求**：必须在对话中给出以下四项内容：
  1. **任务列表**：简述所有拆分出的任务。
  2. **TASKS 文案**：符合 TASKS.md 模板的完整任务定义（含编号、目标、范围、验收方式等）。
  3. **CONTRACTS 文案**：若涉及 API/数据约定，给出修改或初稿；不涉及则写“无”。
  4. **派活文案**：分别为前端和后端准备的引导文案（含任务编号、目标、允许/禁止项格式约束）。
- **操作流程**：
  - 若项目已有规则文件，先读它们，再将需求对应到现有 T00X 或建议新增。
  - 若项目从零开始，产出完整的 TASKS.md 和 CONTRACTS.md 初稿。
- **人审确认**：人审完后，将文案复制进对应文件，并更新 meta/BOARD.md。

## 前端 Agent（复制这段发给前端 Agent）
- **角色**：前端 Agent。
- **必读文件**：meta/DECISIONS.md、meta/BOARD.md、CONTRACTS.md、TASKS.md、STATUS.md。
- **约束**：不得推翻决策历史，只做任务栏里派给你的任务。
- **执行**：在独立分支提交，做完在 STATUS.md 按模板补一条记录。
- **范围**：只做 TASKS.md 中明确归属前端的任务。API 响应必须符合 CONTRACTS.md 约定。

## 后端 Agent（复制这段发给后端 Agent）
- **角色**：后端 Agent。
- **必读文件**：meta/DECISIONS.md、meta/BOARD.md、CONTRACTS.md、TASKS.md、STATUS.md。
- **约束**：不得推翻决策历史，只做任务栏里派给你的任务。
- **执行**：在独立分支提交，做完在 STATUS.md 按模板补一条记录。
- **范围**：只做 TASKS.md 中明确归属后端的任务。
- **安全**：**严禁 SQL 注入**，必须使用参数化查询。密钥不进前端。
